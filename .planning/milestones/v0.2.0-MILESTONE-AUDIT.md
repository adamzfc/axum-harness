## Critical Gaps

### 1. AUTH-01: Google Login Not Wired to Adapter

**Phase:** 04-minimal-feature-implementation
**Status:** BLOCKED

`packages/adapters/auth/google/src/lib.rs` contains a full `GoogleAuthAdapter` implementation (529 lines). `packages/features/auth/src/lib.rs` defines `AuthService` trait. But `packages/adapters/hosts/tauri/src/commands/auth.rs` still contains the complete inline OAuth/PKCE implementation (`start_oauth`, `handle_oauth_callback`, `get_session`) and never imports or calls `GoogleAuthAdapter` or `feature_auth::AuthService`.

**Reproducible check:**
```
rg -n "GoogleAuthAdapter|feature_auth::AuthService" packages/adapters/hosts/tauri/src/commands/auth.rs
# → no matches
```

**Fix required:** Wire Tauri auth commands to use `GoogleAuthAdapter` through `feature-auth::AuthService`. Remove or minimize duplicate OAuth logic in `auth.rs`.

### 2. AGENT-DEV-01: Phase 5 Unverified, Prompts Missing

**Phase:** 05-agent-friendly
**Status:** UNSATISFIED (orphaned)

- No VERIFICATION.md exists for Phase 5
- `.agents/prompts/` directory is empty (only `.gitkeep`)
- Requirement AGENT-DEV-01 specifies "skills、prompts、playbooks、rubrics" — prompts are missing
- Playbooks (create-feature, update-contracts) and rubrics (code-review, task-completion, boundary-compliance) exist and are substantive

**Fix required:** Create prompts content, generate VERIFICATION.md for Phase 5.

### 3. CONTRACT-02: Generated Types Never Consumed

**Phase:** 02-contracts-typegen
**Status:** PARTIAL

Typegen pipeline mechanically works: `moon run repo:typegen` → `cargo test` generates `.ts` → copies to `apps/client/web/app/src/lib/generated/`. But zero imports of these generated types exist in any frontend `.svelte` or `.ts` file. Frontend defines its own inline types (`UserProfile`, `AuthSession` in `ipc/auth.ts`).

**Impact:** If a contract DTO changes, generated `.ts` updates but no frontend code breaks because nothing imports them. Drift check passes but provides false confidence.
