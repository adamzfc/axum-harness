<task type="auto">
  <name>Task 1: 在 contracts/auth 中添加 UserProfile 类型并生成</name>
  <files>packages/contracts/auth/src/lib.rs, apps/client/web/app/src/lib/generated/auth/UserProfile.ts</files>
  <action>
    当前前端 ipc/auth.ts 中定义了 UserProfile { email, name, picture, sub }，
    但 contracts/auth/src/lib.rs 中没有对应类型。
    
    在 packages/contracts/auth/src/lib.rs 中添加：
    ```rust
    /// User profile information from OAuth provider.
    #[derive(Debug, Clone, Serialize, Deserialize, ToSchema, TS)]
    #[ts(export, export_to = "auth/")]
    pub struct UserProfile {
        pub email: String,
        pub name: String,
        pub picture: String,
        pub sub: String,
    }
    ```
    
    同时添加对应的测试：
    ```rust
    #[test]
    fn export_user_profile() {
        UserProfile::export().unwrap();
    }
    ```
    
    运行 moon run repo:typegen 生成 TypeScript 类型到 frontend/generated/auth/UserProfile.ts。
    
    注意：UserSession 中的 user_sub 字段与 UserProfile 的 sub 字段语义相同但命名不同，
    这是设计决策——UserSession 是精简的会话标识，UserProfile 是完整的用户信息。
    保持两者共存，前端根据场景使用。
  </action>
  <verify>
    <automated>ls apps/client/web/app/src/lib/generated/auth/UserProfile.ts && grep "export type UserProfile" apps/client/web/app/src/lib/generated/auth/UserProfile.ts</automated>
  </verify>
  <done>UserProfile 类型从 contracts 生成到 frontend/generated/auth/，可导入使用</done>
</task>
