# Phase 14.1: Deferred items — cargo 编译拖慢 E2E 与 E2E 未全绿修复 - Context

**Gathered:** 2026-04-07  
**Status:** Ready for planning  
**Source:** Default discuss conclusions (deferred-items + Phase 14 constraints)

<domain>
## Phase Boundary

在 Phase 14 已完成 WDIO 退场与 tauri-playwright 单栈后，修复遗留阻塞：

1. `apps/client/web/app` 的 web Playwright E2E 在 tenant 初始化阶段出现 `401` / `ECONNREFUSED`，导致全仓 E2E 不能全绿。
2. E2E 运行存在高启动成本（重复 cargo 编译/启动 runtime_server），导致迭代速度过慢。
3. E2E 预检不稳定（端口占用、类型文件同步时序），导致假红。

本阶段不新增业务功能，不改变业务语义，仅修复 E2E 通道稳定性与执行效率。

</domain>

<decisions>
## Implementation Decisions

### Locked Decisions

- **D-01:** 修复 `tenant init failed: status=401`，必须让 web E2E 的 tenant 初始化请求满足后端鉴权契约（与现有测试 JWT 语义一致）。
- **D-02:** 修复 `ECONNREFUSED 127.0.0.1:3001`，E2E 执行前必须具备可重复的 runtime_server 可用性保障（ready check + 按需启动/复用）。
- **D-03:** 优先减少重复 cargo 编译/冷启动成本，建立可复用 runtime 进程或预构建策略，降低 full E2E 迭代耗时。
- **D-04:** 增加 deterministic preflight（端口/进程/类型生成）以减少假失败，不允许仅靠“重试碰运气”通过。
- **D-05:** 保持 Phase 14 的迁移决策：WDIO 已退场，活动 E2E 门禁仅为 web Playwright + tauri-playwright（沿用 D-06-override）。
- **D-06:** 不得通过修改产品业务行为来迁就测试，修复必须落在测试编排、fixture、启动流程和契约对齐层。
- **D-07:** 验收口径保持“全仓活动 E2E 全绿 + 可诊断证据”。

### the agent's Discretion

- runtime_server 复用实现细节（独立脚本、global setup、共享 helper 的组织方式）。
- preflight 放置位置（moon task、Playwright global setup、Just 包装）与失败文案。
- 诊断输出目录命名，只要满足可追踪、可下载、可复现。

</decisions>

<specifics>
## Specific Ideas

- 若 runtime_server 已运行，E2E 不应重复拉起新实例。
- 若 runtime_server 未运行，自动启动路径应尽量使用已构建产物，避免每次冷编译。
- preflight 需显式检查并处理：端口冲突、缺失 typegen 产物、API ready 状态。

</specifics>
