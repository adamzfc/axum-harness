<task type="auto" tdd="false">
  <name>Task 2: 创建 8 个 crate 骨架</name>
  <files>crates/domain/Cargo.toml, crates/domain/src/lib.rs, crates/application/Cargo.toml, crates/application/src/lib.rs, crates/shared_contracts/Cargo.toml, crates/shared_contracts/src/lib.rs, crates/github_adapter/Cargo.toml, crates/github_adapter/src/lib.rs, crates/persistence_sqlite/Cargo.toml, crates/persistence_sqlite/src/lib.rs, crates/notification_adapter/Cargo.toml, crates/notification_adapter/src/lib.rs, crates/runtime_tauri/Cargo.toml, crates/runtime_tauri/src/lib.rs, crates/runtime_server/Cargo.toml, crates/runtime_server/src/lib.rs</files>
  <action>
    为每个 crate 创建 `Cargo.toml` 和 `src/lib.rs`。

    **依赖分层规则（来自 docs/03 §4）：**
    - domain: 纯领域，无外部依赖
    - shared_contracts: 纯 DTO/enum，无外部依赖
    - application: 依赖 domain + shared_contracts
    - github_adapter: 依赖 domain + shared_contracts + octocrab + reqwest
    - persistence_sqlite: 依赖 domain + shared_contracts + rusqlite
    - notification_adapter: 依赖 domain + shared_contracts + tauri-plugin-notification
    - runtime_tauri: 依赖所有 crates + tauri
    - runtime_server: 依赖 domain + shared_contracts + axum（future，当前空壳）

    每个 Cargo.toml 格式：
    ```toml
    [package]
    name = "crate_name"
    version = "0.1.0"
    edition = "2021"

    [dependencies]
    # workspace 依赖用 { workspace = true }
    ```

    每个 `src/lib.rs` 内容：
    ```rust
    //! [crate_name] — [一句话描述]
    ```

    具体 crate 依赖：
    1. `domain` — 无依赖
    2. `shared_contracts` — 无依赖，依赖 serde = { workspace = true }
    3. `application` — domain = { path = "../domain", workspace = true }（注意：path 路径不支持 workspace = true，用 path 指向 + 从 workspace 继承版本）
       实际上 Cargo workspace 用 `dep_name = { workspace = true }` 即可，path 由 workspace members 自动解析。
       但是 workspace 内部 crate 间依赖不需要在根 Cargo.toml 声明，直接用 `dep = { path = "../dep" }`。
       **正确做法：** 在 crate 的 Cargo.toml 中直接写 `domain = { path = "../domain" }`，版本不需要（path 依赖不走 registry）。
    4. `github_adapter` — domain, shared_contracts, octocrab, reqwest, serde
    5. `persistence_sqlite` — domain, shared_contracts, rusqlite, serde_rusqlite, tracing
    6. `notification_adapter` — domain, shared_contracts, tauri-plugin-notification, tracing
    7. `runtime_tauri` — domain, application, github_adapter, persistence_sqlite, notification_adapter, shared_contracts, tauri, tokio, tracing
    8. `runtime_server` — domain, shared_contracts（空壳，axum 在 future 引入）
  </action>
  <verify>
    <automated>ls crates/*/Cargo.toml | wc -l 应输出 8，ls crates/*/src/lib.rs | wc -l 应输出 8</automated>
  </verify>
  <done>8 个 crate 各有 Cargo.toml + src/lib.rs，依赖分层符合架构规范</done>
</task>
