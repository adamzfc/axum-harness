<task type="auto" tdd="false">
  <name>Task 1: 创建 moon 任务定义</name>
  <files>moon.yml, apps/desktop-ui/moon.yml, crates/domain/moon.yml, crates/application/moon.yml, crates/shared_contracts/moon.yml</files>
  <action>
    创建 moon 任务配置文件：

    1. 根级 `moon.yml`（任务继承基线）:
    ```yaml
    tasks:
      build:
        command: 'cargo build'
        inputs:
          - '@globs(sources)'
          - 'Cargo.toml'
        platform: 'system'

      test:
        command: 'cargo test'
        inputs:
          - '@globs(sources)'
        platform: 'system'

      check:
        command: 'cargo check'
        inputs:
          - '@globs(sources)'
          - 'Cargo.toml'
        platform: 'system'

      lint:
        command: 'cargo clippy -- -D warnings'
        inputs:
          - '@globs(sources)'
        platform: 'system'

      format:
        command: 'cargo fmt --all'
        inputs:
          - '@globs(sources)'
        platform: 'system'
    ```

    2. `apps/desktop-ui/moon.yml`（前端覆盖）:
    ```yaml
    tasks:
      dev:
        command: 'pnpm dev'
        local: true
        platform: 'system'

      build:
        command: 'pnpm build'
        inputs:
          - 'src/**/*'
          - 'svelte.config.js'
          - 'vite.config.ts'
          - 'package.json'
        platform: 'system'

      check:
        command: 'pnpm check'
        inputs:
          - 'src/**/*'
          - 'tsconfig.json'
        platform: 'system'

      lint:
        command: 'pnpm lint'
        inputs:
          - 'src/**/*'
        platform: 'system'

      format:
        command: 'pnpm format'
        inputs:
          - 'src/**/*'
        platform: 'system'

      # Tauri 相关
      tauri:build:
        command: 'cargo build'
        args:
          - '--manifest-path'
          - 'src-tauri/Cargo.toml'
        inputs:
          - 'src-tauri/**/*'
          - 'src-tauri/Cargo.toml'
        platform: 'system'
    ```

    3. `crates/domain/moon.yml`（crate 示例，使用根级继承）:
    ```yaml
    tasks:
      build:
        command: 'cargo build -p domain'
      test:
        command: 'cargo test -p domain'
      check:
        command: 'cargo check -p domain'
      lint:
        command: 'cargo clippy -p domain -- -D warnings'
      format:
        command: 'cargo fmt'
    ```

    4. `crates/application/moon.yml` 和 `crates/shared_contracts/moon.yml`：同上模式，替换 `-p` 参数为对应 crate name。
  </action>
  <verify>
    <automated>test -f moon.yml && test -f apps/desktop-ui/moon.yml && echo "MOON_TASKS_OK"</automated>
  </verify>
  <done>moon.yml 任务定义存在，前端和主要 crate 各有项目级 moon.yml</done>
</task>

<task type="auto" tdd="false">
  <name>Task 2: 创建 GitHub Actions CI</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    创建 `.github/workflows/ci.yml`：
    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      ci:
        name: Lint, Check, Test
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - uses: moonrepo/setup-moon@v2

          - name: Run CI
            run: moon ci
    ```
  </action>
