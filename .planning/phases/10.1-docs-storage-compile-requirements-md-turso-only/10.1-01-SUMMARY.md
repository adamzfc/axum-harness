# Phase 10.1 Plan 01: Turso-only 默认启动收敛 Summary

**runtime_server 默认启动链路已收敛为 Turso 文件持久化，并具备路径来源可观测与初始化失败即退出的 fail-fast 契约。**

## Performance

- **Completed:** 2026-04-08T14:14:00Z
- **Tasks:** 2
- **Files modified:** 7

## Accomplishments
- Task 1 完成 Turso-only 默认配置收敛：provider 默认 Turso、默认路径固定文件库、拒绝 memory URL。
- Task 2 完成配置化启动：`main.rs` 不再走 `new_dev()`，改为 `new_with_config(config)` 并输出 provider/path/source。
- 启动初始化包含父目录创建、tenant/counter/agent 迁移与错误即返回，且相关 HTTP E2E/tracing 测试完成适配。

## Task Commits

Each task was committed atomically:

1. **Task 1: 固化 runtime_server Turso-only 默认配置与路径来源契约** - `472bec9` (feat)
2. **Task 2: 切换服务端启动入口为配置化 Turso 初始化并 fail-fast** - `322b58b` (feat)

## Files Created/Modified
- `servers/api/src/config.rs` - 默认 provider/url 改为 Turso 文件路径，新增 `resolved_db_path` 与来源判定。
- `servers/api/src/state.rs` - 新增启动 DB 元数据解析、目录创建与 embedded Turso 迁移的 fail-fast 初始化流程。
- `servers/api/src/main.rs` - 启动改为 `new_with_config`，记录 provider/path/source 结构化日志。
- `servers/api/src/routes/tenant.rs` - tenant 初始化逻辑切换至 embedded Turso。
- `servers/api/src/routes/health.rs` - readiness 健康检查切换至 embedded Turso `health_check`。
- `servers/api/tests/http_e2e_test.rs` - 测试状态构建改为 Turso 文件库并对齐迁移策略。
- `servers/api/tests/tracing_test.rs` - tracing 测试状态改为 Turso 文件库。

## Decisions Made
- 保留 `AppState` 结构稳定性（含兼容字段），但默认运行路径完全走 Turso 初始化，避免扩大结构性重构范围。
- 在启动和测试都增加防御性 tenant 表迁移 SQL，确保底层执行语义差异时仍满足 fail-fast 与可用性。
