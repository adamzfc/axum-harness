# Phase 10.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| process→filesystem | Startup writes/creates DB path — path injection & permission risk | File path string (database URL) |
| env→config | Environment variables override DB path — untrusted input | APP_DATABASE__URL, NATIVE_TAURI_TURSO_DB_PATH |
| env→native runtime | Env override may target unsafe locations | NATIVE_TAURI_TURSO_DB_PATH |
| repo config→build graph | Cargo features determine which adapters compile | Feature flags, dependency declarations |
| CI workflow→release gate | CI determines release evidence trust | Workflow steps, artifact names |
| test harness→auth stub | Test identity at controlled auth boundary | Tenant pair fixtures, auth headers |
| tenant A→tenant B data path | Cross-tenant read/write is primary isolation risk | API routes, DB queries |
