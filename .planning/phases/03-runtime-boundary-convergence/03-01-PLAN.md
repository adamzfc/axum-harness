<objective>
Migrate port implementations from host crates to packages/adapters/storage/ as independent adapter crates.

Purpose: Port implementations (TenantAwareSurrealDb, TursoDb, EmbeddedLibSql) currently live inside servers/api and native-tauri. This violates hexagonal architecture — adapters must be independent crates that host apps depend on, not embedded within host code.
Output: Two new adapter crates (storage_surrealdb, storage_libsql) with all port implementations, tests, and migration functions. Original host crates updated to import from adapters.
</objective>
